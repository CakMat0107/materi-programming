Spesifikasi Kasus 01: Beasiswa Tahfizh

Tingkat: Sederhana

Sebuah pondok pesantren membuka Beasiswa Tahfizh dengan aturan kualifikasi sebagai berikut:

  • Jika hafalan ≥ 10 Juz DAN nilai rata-rata ≥ 85.0 → Output: "Lolos Beasiswa Penuh"
  • Jika hafalan ≥ 5 Juz DAN nilai rata-rata ≥ 75.0 → Output: "Lolos Beasiswa Parsial"
  • Selain dari kondisi di atas → Output: "Tidak Lolos Beasiswa"

Petunjuk Taktis:

Gunakan operator logika and untuk menggabungkan dua syarat (hafalan DAN nilai) di dalam pernyataan if dan elif.

Spesifikasi Kasus 02: Diskon Toko Buku Kitab

Tingkat: Menengah

Toko Kitab memberikan potongan harga berdasarkan total belanjaan dan status keanggotaan santri:

  • Jika total belanja ≥ Rp 200.000 DAN member True → Diskon 20%
  • Jika total belanja ≥ Rp 200.000 DAN member False → Diskon 10%
  • Jika total belanja < Rp 200.000 DAN member True → Diskon 5%
  • Selain dari itu → Diskon 0%

Klik "Uji Semua Test Case" untuk memverifikasi logika kamu.
Petunjuk Taktis:

Kamu dapat menggunakan ekspresi boolean is_member secara langsung atau menggunakan operator not is_member untuk memeriksa status non-member.

Spesifikasi Kasus 03: Poin Pelanggaran Santri

Tingkat: Kompleks

Sistem Kedisiplinan Pesantren mengklasifikasikan tindakan berdasarkan akumulasi poin pelanggaran:

  • Poin ≥ 100 → Output: "SP3 - Pemanggilan Orang Tua"
  • Poin 50 - 99 → Output: "SP2 - Peringatan Keras"
  • Poin 20 - 49 → Output: "SP1 - Teguran Lisan"
  • Poin < 20 → Output: "Sangat Baik"
